# test25：constexpr 容器安全查找与哈希分发边界

## 目标
- 补齐 `test12` 第27章的工程边界：避免“查不到 key 仍解引用”的风险。
- 演示“哈希 switch 分发”在工程中应做二次字符串确认，避免碰撞误命中。

## 运行
```bash
g++ -std=c++17 -O2 -o test25 test25.cpp
./test25
```

## 核心点
- `ConstexprMap::get_copy()` 返回 `std::optional<V>`，未命中时安全返回空值。
- `switch(fnv1a(cmd))` 命中后再 `cmd == "literal"` 做二次校验。
- 该模式保留了编译期常量分发优势，也显式覆盖了哈希碰撞边界。

## 输出解读
- `404 -> Not Found`：命中 key，返回值正常。
- `302 -> <missing>`：未命中 key，安全空返回，无未定义行为。
- `stat -> 未知`：哈希未匹配到有效命令，落入默认分支。

## 适用场景
- 配置码表、协议码表、状态码映射等“小而固定”的查找表。
- 命令分发/脚本指令解析中希望利用编译期哈希但仍要求稳健性的场景。
