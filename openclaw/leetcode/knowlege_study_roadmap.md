# 1. 写在最前面：给零基础同学的话

## 1.1 你可能有这些疑问

> **Q: 我完全没有编程基础，能学 LeetCode 吗？**
> A: 可以！但需要先学 C++ 基础语法（2-4 周），本路线会告诉你需要什么基础。

> **Q: 刷多少题才够？**
> A: **质量 > 数量**。200 题精刷 > 500 题水过。理解一道题的多种解法比做十道新题更重要。

> **Q: 看不懂题解怎么办？**
> A: 正常！先看对应专题的知识文档理解原理，再看题解。每个专题都有从原理到代码的完整讲解。

> **Q: 多久能看到效果？**
> A: 坚持 2 周你会觉得没那么难了；1 个月后能独立解 Easy；3 个月后能解大部分 Medium。

## 1.2 心态准备

```
✅ 正确心态：
 • 看不懂解法 → 很正常，回去看知识文档
 • 一道题想了 30 分钟没思路 → 看题解学习，不丢人
 • 过了一周忘了 → 很正常，定期复习就好
 • 一天只能做 1 题 → 没关系，贵在坚持

❌ 错误心态：
 ✗ "别人都会为什么我不会" → 每个人起点不同
 ✗ "这题一定要自己想出来" → 学习阶段看解法学习更高效
 ✗ "我要一天刷 10 道" → 3 道搞懂胜过 10 道水过
 ✗ "Easy 太简单不想做" → Easy 是练手感和建立信心的
```

# 9. 正确的刷题方法论

## 9.1 一道题的正确打开方式

```
Step 1: 审题（3 分钟）
  • 读题 2-3 遍，确保理解
  • 看数据范围 → 反推时间复杂度
  • 看示例，手动验证理解

Step 2: 想思路（10-15 分钟）
  • 暴力法能想到就先想暴力法
  • 套用决策图（见附录详解文档）匹配模式
  • 如果 15 分钟没思路 → 看提示/标签

Step 3: 编码（15-20 分钟）
  • 先写伪代码框架
  • 再填充细节
  • 处理边界条件

Step 4: 调试 & 提交（5 分钟）
  • 用示例手动走一遍代码
  • 处理边界用例（空、单元素等）
  • 提交

Step 5: 学习最优解（5-10 分钟）  ← 最重要！
  • 看排名靠前的解法
  • 和自己的比较，学习更优思路
  • 记录到笔记模板中
```

## 9.2 复习策略：艾宾浩斯遗忘曲线

```
做完一道题后的复习时间点：
  • 第 1 天后：快速回忆解法（不看代码）
  • 第 3 天后：重做一遍
  • 第 7 天后：只看题目，回忆思路
  • 第 14 天后：完全独立重做
  • 第 30 天后：如果还记得，这道题就真正掌握了
```

## 9.3 刷题三遍法

| 遍数             | 目标 | 方法                          |
| ---------------- | ---- | ----------------------------- |
| **第一遍** | 学习 | 不会就看题解，理解算法思路    |
| **第二遍** | 巩固 | 隔 3-7 天重做，尽量独立完成   |
| **第三遍** | 内化 | 隔 2-4 周重做，确保能快速写出 |

# 10. 从失败中学习：常见误区

## 10.1 刷题误区

| 误区                   | 问题                   | 正确做法                    |
| ---------------------- | ---------------------- | --------------------------- |
| **贪多求快**     | 一天做 10 题但都不理解 | 一天 2-3 题，每题搞透       |
| **只做 Easy**    | 停留在舒适区           | Easy 练手感后尽快做 Medium  |
| **不看题解**     | 一道题死磕 2 小时      | 20 分钟没思路就看题解学习   |
| **只看题解不写** | 觉得看懂了就行         | 一定要动手写代码 AC         |
| **不做笔记**     | 每道题做完就忘         | 用笔记模板记录关键思路      |
| **随机刷题**     | 东一题西一题没有体系   | 按专题系统刷                |
| **不复习**       | 做过的题回头不认识     | 按遗忘曲线定期复习          |
| **追求一次 AC**  | 没有 bug 不正常        | 调试是正常的，从 bug 中学习 |

## 10.2 学习误区

| 误区                     | 问题           | 正确做法             |
| ------------------------ | -------------- | -------------------- |
| **先学完所有理论** | 学了很久不动手 | 理论+实践交替进行    |
| **跳过基础**       | 直接学 DP/图论 | 按路线从基础开始     |
| **只看视频**       | 被动学习效率低 | 看完视频立刻动手实践 |
