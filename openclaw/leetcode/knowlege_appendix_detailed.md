# 1. 时间复杂度完全指南

## 1.1 什么是时间复杂度？

时间复杂度描述的是**算法执行所需操作次数**与输入规模 $n$ 之间的增长关系，用**大 O 符号** $O(\cdot)$ 表示。

> **核心原则**：只关注最高阶项，忽略常数因子和低阶项。

### 读法示意

| 写法 | 含义 | 直觉理解 |
|------|------|----------|
| $O(1)$ | 常数时间 | 不管数据多大，一步搞定 |
| $O(\log n)$ | 对数时间 | 每次砍掉一半（二分） |
| $O(n)$ | 线性时间 | 看一遍就完事 |
| $O(n \log n)$ | 线性对数 | 排个序的成本 |
| $O(n^2)$ | 平方时间 | 两层 for 循环 |
| $O(n^3)$ | 立方时间 | 三层 for 循环 |
| $O(2^n)$ | 指数时间 | 枚举所有子集 |
| $O(n!)$ | 阶乘时间 | 枚举所有排列 |

## 1.2 常见复杂度增长对比

假设计算机每秒执行 $10^8$ 次操作：

| $n$ | $O(\log n)$ | $O(n)$ | $O(n \log n)$ | $O(n^2)$ | $O(2^n)$ | $O(n!)$ |
|-----|-------------|--------|---------------|----------|----------|---------|
| 10 | 3 | 10 | 33 | 100 | 1024 | 362万 |
| 100 | 7 | 100 | 664 | 10000 | $10^{30}$ | 爆炸 |
| 1000 | 10 | 1000 | 9966 | $10^6$ | 爆炸 | 爆炸 |
| $10^5$ | 17 | $10^5$ | $1.7 \times 10^6$ | $10^{10}$ ❌ | 爆炸 | 爆炸 |
| $10^6$ | 20 | $10^6$ | $2 \times 10^7$ | $10^{12}$ ❌ | 爆炸 | 爆炸 |

> **⚠️ 关键阈值**：当操作次数超过 $10^8$（约 1 秒），就会 TLE（超时）。

## 1.3 如何分析代码的时间复杂度？

### 规则 1：循环次数

```cpp
// O(n)
for (int i = 0; i < n; i++) { ... }

// O(n²)
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++) { ... }

// O(n log n)
for (int i = 0; i < n; i++)
    for (int j = 1; j < n; j *= 2) { ... }

// O(log n) — 每次减半
while (n > 0) n /= 2;
```

### 规则 2：递归

- 一般用**递归树**或**主定理**分析
- 二分递归：$T(n) = T(n/2) + O(1) \Rightarrow O(\log n)$
- 归并排序：$T(n) = 2T(n/2) + O(n) \Rightarrow O(n \log n)$
- 暴力子集：$T(n) = 2T(n-1) \Rightarrow O(2^n)$

### 规则 3：均摊分析

某些操作单次可能是 $O(n)$，但 $n$ 次操作总共 $O(n)$，均摊每次 $O(1)$。

经典例子：
- `vector::push_back` — 偶尔扩容 $O(n)$，均摊 $O(1)$
- 并查集路径压缩 — 均摊 $O(\alpha(n)) \approx O(1)$
- 单调栈 — 每个元素最多入栈出栈各一次，总共 $O(n)$

# 2. 空间复杂度完全指南

## 2.1 什么算进空间复杂度？

| 计入 | 不计入 |
|------|--------|
| 你创建的数组、哈希表 | 输入数据本身 |
| 递归调用栈深度 | 输出结果（看题目要求） |
| 额外数据结构（堆、栈等） | 常数个变量 |

## 2.2 常见空间复杂度

| 空间 | 场景 |
|------|------|
| $O(1)$ | 原地操作，只用几个变量 |
| $O(\log n)$ | 递归二分的栈深度 |
| $O(n)$ | 缓存数组、哈希表、队列 |
| $O(n^2)$ | 二维 DP 表、邻接矩阵 |
| $O(n \cdot m)$ | 二维字符串 DP 表 |

## 2.3 空间优化常见手段

| 手段 | 思路 | 典型应用 |
|------|------|----------|
| **滚动数组** | 只保留上一行/上两行 DP 数据 | 二维 DP → 一维 |
| **原地修改** | 不建新数组，在原数组上标记 | 矩阵搜索、数组去重 |
| **位压缩** | 一个 int 存 32 个 bool | 状态压缩 DP |
| **拓扑序遍历** | 不存访问数组，利用输入结构 | 特殊图问题 |
| **Morris 遍历** | 不用栈遍历二叉树 | 中序遍历 O(1) 空间 |

## 3.2 什么时候选哪个？

```
需要频繁随机访问？
├── 是 → vector / 数组
│   ├── 需要两端高效插删？ → deque
│   └── 中间频繁插入？ → list (链表)
│
需要快速查找/判重？
├── 不关心顺序 → unordered_set / unordered_map  O(1)
└── 需要有序 → set / map  O(log n)
│
需要取最值？
├── 只需要最大/最小 → priority_queue (堆)
└── 需要任意区间最值 → 线段树
│
需要前缀信息（和、积等）？
├── 静态不修改 → 前缀和数组
├── 单点修改 + 区间查询 → 树状数组 BIT
└── 区间修改 + 区间查询 → 线段树
│
需要处理连通性？
├── 静态连通 → DFS / BFS
└── 动态合并 → 并查集
│
需要前缀匹配？
└── Trie 字典树
```

## 3.3 C++ STL 容器性能实测要点

| 容器 | 注意 |
|------|------|
| `unordered_map` | 最坏退化 $O(n)$，被恶意构造数据卡。可用自定义哈希防卡 |
| `map` | 稳定 $O(\log n)$，但常数较大 |
| `vector` | 预先 `reserve()` 可避免多次扩容 |
| `deque` | 内存不连续，缓存不如 vector |
| `priority_queue` | 默认大根堆。小根堆用 `greater<int>` |
| `set/multiset` | `multiset` 的 `erase(val)` 会删所有！用 `erase(find(val))` 只删一个 |

# 4. 排序算法复杂度全表

| 排序算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 | 适用场景 |
|---------|------|------|------|------|------|----------|
| 冒泡排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✅ | 教学用 |
| 选择排序 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ❌ | 教学用 |
| 插入排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✅ | 近乎有序的小数组 |
| 希尔排序 | $O(n \log n)$ | 约 $O(n^{1.3})$ | $O(n^2)$ | $O(1)$ | ❌ | 中等规模 |
| **归并排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | ✅ | 需要稳定排序、逆序对 |
| **快速排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | ❌ | 通用排序（STL 默认） |
| **堆排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(1)$ | ❌ | 空间严格限制、Top K |
| 计数排序 | $O(n+k)$ | $O(n+k)$ | $O(n+k)$ | $O(k)$ | ✅ | 值域小的整数 |
| 桶排序 | $O(n+k)$ | $O(n+k)$ | $O(n^2)$ | $O(n+k)$ | ✅ | 均匀分布数据 |
| 基数排序 | $O(d \cdot n)$ | $O(d \cdot n)$ | $O(d \cdot n)$ | $O(n+k)$ | ✅ | 多关键字/大范围整数 |

> $k$ = 值域大小/桶数，$d$ = 最大位数。

### 面试常考要点

1. **快排为什么平均 $O(n \log n)$ 最坏 $O(n^2)$？**  
   随机选 pivot 平均均匀分割 → $T(n) = 2T(n/2) + O(n)$。最坏每次只排掉一个元素。

2. **什么时候用归并而不是快排？**  
   - 需要**稳定排序**（相同键保持原始顺序）
   - 统计**逆序对**
   - 链表排序

3. **堆排序优缺点？**  
   优：原地 $O(1)$ 空间，最坏 $O(n \log n)$。缺：常数大、缓存不友好。

4. **`std::sort` 用的什么？**  
   Introsort = 快排 + 堆排 + 插入排序混合。

## 5.2 复杂度推导速记口诀

```
一个 for      → O(n)
嵌套 k 层 for → O(n^k)
每次砍一半    → O(log n)
排序打底      → O(n log n)
枚举子集      → O(2^n)
枚举排列      → O(n!)
树/图遍历     → O(V + E)
DP = 状态数 × 每个状态转移代价
```

# 7. LeetCode 常见时间限制与复杂度估算

## 7.1 根据 $n$ 的范围反推可用算法

这是 LeetCode 刷题中**最实用的技巧之一**！看到数据范围就能知道该用什么复杂度的算法。

| $n$ 的范围 | 可接受的最大复杂度 | 常用算法 |
|-----------|-------------------|---------|
| $n \leq 10$ | $O(n!)$ 或 $O(n \cdot 2^n)$ | 暴力、回溯、状压 DP |
| $n \leq 20$ | $O(2^n)$ | 回溯、状态压缩 DP、暴力枚举子集 |
| $n \leq 30$ | $O(2^{n/2})$ | Meet in the Middle 折半枚举 |
| $n \leq 100$ | $O(n^3)$ | 区间 DP、Floyd |
| $n \leq 1000$ | $O(n^2)$ | 朴素 DP、暴力搜索 |
| $n \leq 5000$ | $O(n^2)$ | 稍宽松的 $n^2$，注意常数 |
| $n \leq 10^5$ | $O(n \log n)$ | 排序、堆、二分、线段树 |
| $n \leq 10^6$ | $O(n)$ 或 $O(n \log n)$ | 线性扫描、前缀和、双指针 |
| $n \leq 10^7$ | $O(n)$ | 纯线性算法 |
| $n \leq 10^9$ | $O(\log n)$ 或 $O(\sqrt{n})$ | 二分、数学 |
| $n \leq 10^{18}$ | $O(\log n)$ | 快速幂、二分、矩阵快速幂 |

> **🎯 超级实用**：刷题时第一件事就是看 Constraints（数据范围），直接反推时间复杂度上限。

## 7.2 实战示例

**例 1**：题目说 $1 \leq n \leq 10^5$
→ 需要 $O(n \log n)$ 或更优 → 排序、二分、线段树方向思考

**例 2**：题目说 $1 \leq n \leq 20$
→ 可以 $O(2^n)$ → 回溯/状压 DP

**例 3**：题目说 $1 \leq n \leq 10^4$，$1 \leq m \leq 10^4$
→ $O(nm) = 10^8$，勉强可以 → 优化常数或降到 $O(n \log m)$

## 8.2 每种模式的万能三句话

> 面试时口述解题思路用：

| 模式 | 万能三句话 |
|------|-----------|
| 滑动窗口 | "用右指针扩大窗口直到满足条件，然后收缩左指针优化结果，整个过程 O(n)。" |
| 双指针 | "排序后用左右指针从两端向中间靠拢，根据大小关系决定移动哪个指针。" |
| BFS | "用队列进行层次遍历，每层处理当前所有节点，适合求最短路径。" |
| 二分 | "确定搜索区间和单调性，每次排除一半，注意边界取 mid 还是 mid±1。" |
| DP | "定义状态含义，找到状态转移方程，初始化 base case，通常可空间压缩。" |
| 回溯 | "在决策树上做深度优先搜索，每一步做选择→递归→撤销选择。" |
| 哈希表 | "用哈希表存已遍历元素，实现 O(1) 查找来消除一层循环。" |
| 栈 | "利用 LIFO 特性处理括号匹配或嵌套结构，遇到配对/满足条件就弹栈。" |
| 堆 | "维护大小为 K 的堆，保证堆顶始终是答案，插入/删除 O(log K)。" |
| 贪心 | "每一步做局部最优选择，证明局部最优能推出全局最优（反证/交换论证）。" |
