## 第一章 基础概念与核心操作

### 1.1 二进制表示

计算机中所有整数都以 **二进制** 存储。一个 32 位 `int` 的补码表示：

```
正数 5:   00000000 00000000 00000000 00000101
正数 -5:  11111111 11111111 11111111 11111011  (补码 = 取反 + 1)
```

**关键知识点**：
- 正数：原码 = 反码 = 补码
- 负数：补码 = 按位取反 + 1 → `-n == ~n + 1`
- `int` 范围：$[-2^{31},\; 2^{31}-1]$，即 $[-2147483648,\; 2147483647]$
- `unsigned int` 范围：$[0,\; 2^{32}-1]$

### 1.2 六大基本位操作

| 操作 | 符号 | 规则 | 常见用途 |
|------|------|------|----------|
| **与 AND** | `&` | 两位都为 1→1，否则→0 | 清零/取指定位/判奇偶 |
| **或 OR** | `\|` | 任一位为 1→1 | 置位（设为1） |
| **异或 XOR** | `^` | 不同→1，相同→0 | 交换/去重/找不同 |
| **取反 NOT** | `~` | 0↔1 翻转 | 配合&清位 |
| **左移** | `<<` | 所有位左移，低位补 0 | 等价于 × 2ⁿ |
| **右移** | `>>` | 算术右移（保留符号位） | 等价于 ÷ 2ⁿ（有符号） |

### 1.3 运算优先级（重要！常踩坑）

```
优先级从高到低:
  ~  (取反)           最高
  << >> (移位)
  &  (与)
  ^  (异或)
  |  (或)             最低
```

> ⚠️ **踩坑警告**: `==` 优先级高于 `&` / `^` / `|`！  
> 写 `if (n & 1 == 0)` 实际等于 `if (n & (1 == 0))`，永远是 `n & 0 = 0`！  
> 正确写法：`if ((n & 1) == 0)`，**一定加括号！**

### 1.4 异或的核心性质（超级重要）

| 性质 | 公式 | 解释 |
|------|------|------|
| 自反 | `a ^ a = 0` | 相同的数异或为 0 |
| 零元 | `a ^ 0 = a` | 与 0 异或不变 |
| 交换律 | `a ^ b = b ^ a` | |
| 结合律 | `(a ^ b) ^ c = a ^ (b ^ c)` | |
| 消去律 | `a ^ b ^ b = a` | 两次异或恢复原值 |

**应用**：数组中所有数异或，成对出现的数互相消除，剩下的就是"落单"的数！

## 第二章 常用位运算技巧百科

### 2.1 获取 / 设置 / 清除 / 翻转指定位

```cpp
// 获取第 k 位（0-indexed，从右数）
int getBit(int n, int k) {
    return (n >> k) & 1;
}

// 将第 k 位设为 1
int setBit(int n, int k) {
    return n | (1 << k);
}

// 将第 k 位设为 0
int clearBit(int n, int k) {
    return n & ~(1 << k);
}

// 翻转第 k 位
int flipBit(int n, int k) {
    return n ^ (1 << k);
}
```

### 2.2 lowbit — 获取最低位的 1

```cpp
int lowbit(int n) {
    return n & (-n);     // 等价于 n & (~n + 1)
}
```

**原理**：`-n = ~n + 1`，取反后最低位的 1 以下全变回来：
```
n    = ...10...0  (最低位的1在某位)
~n   = ...01...1
~n+1 = ...10...0  (进位到最低位的1)
n & (-n) = 00...10...0  (只保留最低位的 1)
```

**应用**：树状数组（BIT）的核心！

### 2.3 消除最低位的 1

```cpp
n = n & (n - 1);   // 消除 n 的最低位 1
```

**原理**：`n - 1` 把最低位的 1 变 0，以下全变 1；`&` 后刚好把那个 1 消掉。
```
n     = ...1 0 0 0
n-1   = ...0 1 1 1
n&n-1 = ...0 0 0 0  ← 最低位的1消失了
```

**应用一**：数 1 的个数（Brian Kernighan 算法）
```cpp
int popcount(int n) {
    int cnt = 0;
    while (n) { n &= n - 1; cnt++; }
    return cnt;
}
```

**应用二**：判断是否为 2 的幂
```cpp
bool isPowerOfTwo(int n) {
    return n > 0 && (n & (n - 1)) == 0;
}
```

### 2.4 判断奇偶

```cpp
if (n & 1)   // 奇数（最低位为 1）
if (!(n & 1)) // 偶数（最低位为 0）
```

### 2.5 不用临时变量交换两个数

```cpp
a ^= b;
b ^= a;  // b = b ^ a ^ b = a
a ^= b;  // a = a ^ b ^ a = b
```

> 注意：当 `a` 和 `b` 是同一个地址时会出错（自己异或自己变 0）！

### 2.6 取绝对值（无分支）

```cpp
int absVal(int n) {
    int mask = n >> 31;   // 正数: 0, 负数: -1(全1)
    return (n ^ mask) - mask;
}
```

### 2.7 取两数的最大值/最小值（无分支）

```cpp
int maxVal(int a, int b) {
    int diff = a - b;
    int sign = (diff >> 31) & 1;  // a<b时为1
    return a - sign * diff;
}
```

> ⚠️ 注意溢出风险，实际中还是用 `std::max` 更安全。

### 2.8 `__builtin` 系列（GCC/Clang 内建函数）

```cpp
__builtin_popcount(n)     // 数 1 的个数
__builtin_clz(n)          // 前导零个数 (count leading zeros)
__builtin_ctz(n)          // 尾随零个数 (count trailing zeros)
__builtin_parity(n)       // 1的个数的奇偶性（奇→1，偶→0）

// 64位版本加 ll 后缀
__builtin_popcountll(n)
__builtin_clzll(n)
__builtin_ctzll(n)
```

> **面试Tips**：GCC 内建函数在竞赛中非常好用，一条 CPU 指令完成。但面试手写可能需要用 `n & (n-1)` 循环方式。

## 第三章 位运算经典模式

### 模式一：异或找落单元素

**核心**：`a ^ a = 0`，所有元素异或，成对消除。

| 题目 | 特征 | 方法 |
|------|------|------|
| LC 136 | 一个元素出现 1 次，其余 2 次 | 全部异或 |
| LC 137 | 一个元素出现 1 次，其余 3 次 | 逐位统计 mod 3 |
| LC 260 | 两个元素出现 1 次，其余 2 次 | 异或+分组 |
| LC 268 | 缺失的数字 | 与下标异或 |

#### LC 136 — 只出现一次的数字

```
[4, 1, 2, 1, 2]
→ 4^1^2^1^2 = 4^(1^1)^(2^2) = 4^0^0 = 4
```

#### LC 260 — 只出现一次的数字 III

两个落单元素 a 和 b：
1. 全部异或得 `xor = a ^ b`（≠0，因为 a≠b）
2. 取 `xor` 的最低位 1（`diff = xor & (-xor)`），这一位 a 和 b 不同
3. 按这一位分成两组，组内各自异或，分别得到 a 和 b

#### LC 137 — 只出现一次的数字 II

每个元素出现 3 次，只有一个出现 1 次。  
**方法**：统计每一位上 1 出现的次数 mod 3：

```cpp
int singleNumber(vector<int>& nums) {
    int result = 0;
    for (int bit = 0; bit < 32; bit++) {
        int sum = 0;
        for (int num : nums)
            sum += (num >> bit) & 1;
        if (sum % 3)
            result |= (1 << bit);
    }
    return result;
}
```

### 模式二：位计数与汉明距离

| 题目 | 描述 |
|------|------|
| LC 191 | 数 n 中 1 的个数（hammingWeight） |
| LC 338 | 0~n 每个数的 1 的个数（DP） |
| LC 461 | 两数汉明距离（异或后数1） |
| LC 477 | 数组总汉明距离（逐位统计） |

#### LC 338 — DP 递推

`dp[i] = dp[i & (i-1)] + 1`  
去掉最低位的 1，+1 即可。

#### LC 477 — 总汉明距离

对每一位，统计有 `c` 个 1 和 `n-c` 个 0 → 贡献 `c * (n-c)` 对。加总 32 位。

### 模式三：状态压缩（Bitmask DP）

用一个整数的二进制位来**表示集合状态**，常用于 N ≤ 20 的全排列/子集问题。

```
集合 {0, 2, 3} → 二进制 1101 → 整数 13
```

**常用操作**：

```cpp
// 遍历所有子集
for (int mask = 0; mask < (1 << n); mask++) { ... }

// 遍历 mask 的所有子集（含自身和空集）
for (int sub = mask; sub > 0; sub = (sub - 1) & mask) { ... }

// 判断第 i 位是否属于集合
if (mask & (1 << i))

// 加入第 i 位
mask | (1 << i)

// 删除第 i 位
mask & ~(1 << i)

// 集合大小
__builtin_popcount(mask)
```

#### LC 78 — 子集

对 n 个元素，枚举 0 到 2ⁿ−1 的所有 mask，每个 mask 就是一个子集。

#### LC 698 — 划分为 K 个相等的子集

`dp[mask]` 表示已选元素集合为 mask 时能否有效划分。

#### LC 943 — 最短超级串

TSP 类型的状压 DP。

### 模式四：位操作实现加/减/乘/除

面试经典：不用 +、-、*、/ 运算符实现四则运算。

#### 加法（LC 371）

```cpp
int getSum(int a, int b) {
    while (b != 0) {
        int carry = (unsigned)(a & b) << 1;  // 进位
        a = a ^ b;                            // 无进位加
        b = carry;
    }
    return a;
}
```

**原理**：
- `a ^ b`：无进位加法结果
- `(a & b) << 1`：进位
- 循环直到无进位

#### 除法（LC 29）

用移位代替除法：从高位到低位尝试减去 `divisor << k`。

### 模式五：位运算与子集枚举

#### 枚举一个数的所有子集

```cpp
for (int sub = mask; sub > 0; sub = (sub - 1) & mask) {
    // sub 是 mask 的一个子集
}
// 别忘了处理空集 sub = 0
```

**时间复杂度**：$O(2^{popcount(mask)})$

#### 枚举大小恰好为 k 的子集 (Gosper's hack)

```cpp
int comb = (1 << k) - 1;
while (comb < (1 << n)) {
    // 处理当前 comb
    int x = comb & (-comb);
    int y = comb + x;
    comb = ((comb & ~y) / x >> 1) | y;
}
```

## 第四章 进阶技巧

### 4.1 Gray 码

相邻两个数只有一位不同。

```cpp
// 第 i 个 Gray 码
int gray(int i) {
    return i ^ (i >> 1);
}
```

| i | 二进制 | Gray | Gray二进制 |
|---|--------|------|------------|
| 0 | 000 | 0 | 000 |
| 1 | 001 | 1 | 001 |
| 2 | 010 | 3 | 011 |
| 3 | 011 | 2 | 010 |
| 4 | 100 | 6 | 110 |
| 5 | 101 | 7 | 111 |
| 6 | 110 | 5 | 101 |
| 7 | 111 | 4 | 100 |

应用：LC 89 — 格雷编码。

### 4.2 位反转

```cpp
uint32_t reverseBits(uint32_t n) {
    uint32_t result = 0;
    for (int i = 0; i < 32; i++) {
        result = (result << 1) | (n & 1);
        n >>= 1;
    }
    return result;
}
```

### 4.3 找不大于 n 的最大 2 的幂

```cpp
int highestOneBit(int n) {
    n |= n >> 1;
    n |= n >> 2;
    n |= n >> 4;
    n |= n >> 8;
    n |= n >> 16;
    return (n + 1) >> 1;
}
```

**原理**：把最高位的 1 以下全部填充为 1，然后 +1 再右移。

### 4.4 快速幂（位运算版）

```cpp
long long quickPow(long long base, long long exp, long long mod) {
    long long result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1)  // 当前位为 1
            result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}
```

将指数拆成二进制位。例如 `3^13 = 3^(1101₂) = 3^8 * 3^4 * 3^1`。  
时间复杂度 $O(\log n)$。

### 4.5 位运算实现乘法

```cpp
int multiply(int a, int b) {
    int result = 0;
    while (b) {
        if (b & 1) result += a;
        a <<= 1;
        b >>= 1;
    }
    return result;
}
```

## 第六章 LC 201 — 数字范围按位与

给定范围 [left, right]，求所有数字按位与的结果。

**核心思想**：结果就是 left 和 right 的**二进制公共前缀**。

```
left  = 5 = 0101
right = 7 = 0111

5 = 0101
6 = 0110
7 = 0111
  & -----
    0100  → 4
```

只要 left ≠ right，最低位一定有 0 和 1 的差异，按位与就是 0。反复右移直到相等。

```cpp
int rangeBitwiseAnd(int left, int right) {
    int shift = 0;
    while (left < right) {
        left >>= 1;
        right >>= 1;
        shift++;
    }
    return left << shift;
}
```

## 第七章 常见面试问题的位运算解法

### 7.1 不用加减乘除做加法

见第四章模式四。关键理解 `异或 = 无进位加法`、`与 + 左移 = 进位`。

### 7.2 判断一个数是否为 4 的幂

```cpp
bool isPowerOfFour(int n) {
    // 1. n > 0
    // 2. n 是 2 的幂（只有一个 1）
    // 3. 这个 1 在偶数位上（0x55555555 = 0101...0101）
    return n > 0 && (n & (n - 1)) == 0 && (n & 0x55555555) != 0;
}
```

### 7.3 用位运算计算字母出现情况

```cpp
// 判断字符串是否所有字符都不同（假设只有小写字母）
bool isUnique(string s) {
    int seen = 0;
    for (char c : s) {
        int bit = 1 << (c - 'a');
        if (seen & bit) return false;
        seen |= bit;
    }
    return true;
}
```

### 7.4 位图（Bitmap）思想

用 1 bit 表示一个状态，32 个状态只用一个 int：

```cpp
class Bitmap {
    vector<int> data;
public:
    Bitmap(int n) : data((n + 31) / 32, 0) {}
    void set(int i) { data[i / 32] |= (1 << (i % 32)); }
    void clear(int i) { data[i / 32] &= ~(1 << (i % 32)); }
    bool get(int i) { return data[i / 32] & (1 << (i % 32)); }
};
```

## 第八章 总结与易错点

### 8.1 知识脑图

```
位运算
├── 基础操作
│   ├── &  |  ^  ~  <<  >>
│   ├── 优先级（必须加括号！）
│   └── 补码表示
├── 常用技巧
│   ├── n & (n-1)    消最低1 → popcount / 判2的幂
│   ├── n & (-n)     取最低1 → BIT树状数组
│   ├── 获取/设置/清除/翻转第k位
│   └── __builtin_xxx 内建函数
├── 经典模式
│   ├── 异或找落单 (LC 136/137/260)
│   ├── 位计数 (LC 191/338/461/477)
│   ├── 状压DP (LC 78/698/526/847)
│   ├── 位实现四则运算 (LC 371/29)
│   └── 公共前缀 (LC 201)
└── 进阶
    ├── Gray码 (LC 89)
    ├── 位反转 (LC 190)
    ├── 快速幂
    ├── Gosper's hack (枚举固定大小子集)
    └── 位图 Bitmap
```

### 8.2 七大易错点

| # | 易错点 | 正确做法 |
|---|--------|----------|
| 1 | 忘加括号：`n & 1 == 0` | `(n & 1) == 0` |
| 2 | 左移溢出：`1 << 31` 是 `INT_MIN` | 用 `1LL << k` 或 `1u << k` |
| 3 | 负数右移补符号位 | 需要逻辑右移时用 `unsigned` |
| 4 | `n & (-n)` 在 n=0 时结果为 0 | 注意边界 |
| 5 | 异或 swap 同一地址 | `a ^= a` 结果为 0！只用于不同变量 |
| 6 | 32位/64位混用 | 涉及大数用 `long long` |
| 7 | 状压时 n > 20 | 2²⁰ = 1M 还行，2²⁵ 已超时 |

### 8.3 面试 FAQ

**Q: 位运算题有什么固定套路？**  
A: 看到"出现一次/两次/三次"→ 异或系列；看到"子集/排列/状态"→ 状压DP；看到"距离/相似度"→ 汉明距离；看到"不用+/-"→ 位加法。

**Q: 状态压缩 DP 的时间复杂度？**  
A: 通常 $O(2^n \times n)$ 或 $O(3^n)$（枚举子集的子集），n 一般 ≤ 20。

**Q: `__builtin_popcount` 可以面试用吗？**  
A: 可以，先写出 `n & (n-1)` 的手动版本，再提一句"实际工程中可用 `__builtin` 或 `std::popcount(C++20)`"更佳。

**Q: 快速幂为什么是位运算范畴？**  
A: 因为它把指数拆成二进制位，`exp & 1` 决定是否乘当前 base，`exp >>= 1` 处理下一位，核心就是位操作。
