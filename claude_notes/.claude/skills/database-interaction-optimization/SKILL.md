# database-interaction-optimization

## 目标

识别数据库访问中的慢查询、锁等待、连接池瓶颈和事务放大问题，并给出优先级清晰的优化建议。

## 适用条件

- 现象表现为查询变慢、锁等待、连接池打满或依赖时延放大
- 改动涉及 SQL、ORM、缓存、事务或数据访问层
- 需要从交互模式而非单条 SQL 风格做审查

## 审查重点

- 索引命中、过滤条件、排序与分页策略
- 事务边界、隔离级别、锁等待与重试
- 批处理、缓存、读写分离与连接池配置
- 服务端关键路径中的数据库放大效应

## 输入期望

- 相关查询、表结构或访问层路径
- 慢查询、锁等待、连接池或延迟现象
- 若有 explain、监控或 trace 证据，建议提供

## 输出要求

- 先给主要瓶颈链路
- 再给索引、SQL、事务、连接池层面的建议
- 说明收益、风险和兼容性影响
- 给出验证方式，如 explain、压测、指标观察

## 协作边界

- 数值或业务语义风险可联动 `risk-check`
- 通用性能排序可联动 `performance-bottleneck-identification`
