# api-design-review

## 目标

识别 API 设计中的契约不清、兼容性破坏、错误语义混乱和调用方迁移风险。

## 适用条件

- 新增或修改接口
- 请求/响应字段、错误码、分页、幂等逻辑发生变化
- 需要从调用方和长期演进角度评审接口

## 审查重点

- 输入输出契约是否清晰且一致
- 版本策略、向后兼容和弃用路径
- 错误码、状态码、异常语义是否稳定
- 分页、过滤、排序、幂等和重试语义

## 输入期望

- 接口说明、调用方背景、兼容性要求
- 相关 schema、示例请求和响应
- 若有历史接口约束，建议一并提供

## 输出要求

- 先给接口设计风险概览
- 再给契约、一致性、兼容性和迁移建议
- 标出调用方影响面
- 给出测试和文档更新建议

## 协作边界

- 编解码和报文边界问题转 `protocol-validate`
- 授权与暴露面问题转 `security-review`
