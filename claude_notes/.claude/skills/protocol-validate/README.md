## 🎯 快速使用

### 1. 重启 Claude Code
修改技能后需要**重启 Claude Code** 使新技能生效。

### 2. 基础使用
```bash
/protocol-validate                     # 验证当前目录下所有协议相关代码
/protocol-validate src/fix/            # 验证 FIX 协议实现
/protocol-validate src/fast/encoder.cpp  # 验证 FAST 编码器
```

### 3. 高级选项
```bash
/protocol-validate --protocol fix      # 指定协议类型
/protocol-validate --version FIX.4.4  # 指定协议版本
/protocol-validate --high-priority    # 只验证高危问题
/protocol-validate --output markdown  # 导出 Markdown 报告
```

## 📊 验证维度

### 1. 消息格式验证
- **字段顺序**: 是否按协议规范排列
- **必填字段**: 是否包含所有必需字段
- **字段类型**: 字段类型是否符合规范
- **字段长度**: 可变长度字段是否合规
- **保留字段**: 保留字段是否正确填充

### 2. 校验和验证
- **计算正确性**: 是否正确计算校验和
- **位置正确性**: 校验和字段位置是否正确
- **算法合规性**: 是否使用规定算法
- **验证机制**: 接收端是否正确验证
- **错误处理**: 校验和错误的处理方式

### 3. 会话管理
- **登录流程**: 登录/登出流程是否正确
- **心跳机制**: 心跳消息处理是否正确
- **序列号管理**: 序列号同步机制
- **重传机制**: 消息重传和Gap Fill
- **会话状态**: 会话状态维护是否正确

### 4. 错误处理
- **拒绝消息**: 拒绝消息处理是否正确
- **序列号问题**: 序列号不连续处理
- **未知消息**: 未知消息类型的处理
- **字段错误**: 字段值错误的处理
- **超时处理**: 各种超时情况的处理

### 5. 性能优化
- **零拷贝**: 是否避免不必要拷贝
- **预分配**: 是否预分配缓冲区
- **批处理**: 是否支持消息批处理
- **内存池**: 是否使用内存池管理
- **编解码效率**: 编解码是否高效

## 🛠️ 扩展方式

### 1. 添加新协议支持
**场景**: 你需要支持 MEMX 协议验证

**步骤**:
1. 创建 `CHECKLISTS/memx_protocol_checklist.md`：
```markdown
# MEMX 协议检查清单

## 必查项
- [ ] 是否正确实现订单类型（市价单、限价单、冰山单）
- [ ] 是否正确处理时间戳（纳秒精度）
- [ ] 是否正确实现撮合逻辑（价格优先、时间优先）
- [ ] 是否正确处理取消和修改订单
- [ ] 是否正确实现行情快照和增量更新

## 高风险模式
| 模式 | 风险 | 修复建议 |
|------|------|----------|
| 时间戳精度不足 | 延迟计算错误 | 使用 std::chrono::nanoseconds |
| 订单簿未按价格排序 | 撮合错误 | 实现红黑树或跳跃表 |
| 未处理订单取消 | 内存泄漏 | 实现订单取消机制 |
```

2. 在 `SKILL.md` 中添加 MEMX 协议识别逻辑

### 2. 集成到工作流
详见完整文档
