# 机器学习模型验证规则清单

## 1. 输入输出验证规则

### 1.1 输入数据验证
- [ ] 输入必须是数值型数据（int, float）
- [ ] 输入维度必须与模型训练时一致
- [ ] 不允许包含空值(NaN, None)
- [ ] 不允许包含无穷大值(Infinity)
- [ ] 输入数据范围应在合理区间内
- [ ] 特征列名必须与模型定义一致
- [ ] 数据类型必须正确（如浮点数、整数）

### 1.2 输出验证
- [ ] 输出必须是数值型数据
- [ ] 输出值范围必须在预期范围内
- [ ] 分类模型输出必须是概率或标签
- [ ] 回归模型输出必须是连续值
- [ ] 时间序列预测输出必须有正确的时间戳
- [ ] 异常检测输出必须是异常分数或标签

## 2. 特征工程验证规则

### 2.1 特征构建
- [ ] 特征构建逻辑必须符合业务逻辑
- [ ] 特征转换必须可逆且一致
- [ ] 特征组合必须有意义
- [ ] 特征衍生必须有明确依据
- [ ] 特征缩放方式必须一致

### 2.2 特征质量
- [ ] 特征相关性不应过高（超过0.9）
- [ ] 特征缺失值比率不应超过5%
- [ ] 特征方差不应过小
- [ ] 特征分布应相对稳定
- [ ] 特征重要性应符合预期

## 3. 模型漂移检测规则

### 3.1 数据分布
- [ ] 训练数据和测试数据分布差异不超过5%
- [ ] 特征分布变化不超过3%
- [ ] 目标变量分布变化不超过3%
- [ ] 异常值比例不应大幅增加

### 3.2 时间序列
- [ ] 时间序列数据必须按时间排序
- [ ] 序列长度必须符合预期
- [ ] 时间间隔必须一致
- [ ] 数据采样频率必须合理

## 4. 公平性检查规则

### 4.1 偏见检测
- [ ] 不同群体间的预测结果差异不超过5%
- [ ] 敏感属性（性别、种族等）影响系数在合理范围内
- [ ] 模型对各群体的准确率差异不超过5%
- [ ] 无明显的歧视性偏差

### 4.2 偏见来源
- [ ] 敏感属性不应直接影响模型决策
- [ ] 模型决策应基于业务相关特征
- [ ] 不应存在间接偏见传播
- [ ] 交叉验证结果应一致

## 5. 性能验证规则

### 5.1 模型性能
- [ ] 准确率/召回率/F1分数在合理范围内
- [ ] AUC曲线面积应大于0.7
- [ ] 模型在训练集和测试集上表现应相近
- [ ] 模型收敛性良好，无震荡现象

### 5.2 稳定性
- [ ] 多次运行结果应一致
- [ ] 模型参数应稳定
- [ ] 预测结果不应过度敏感于输入微小变化
- [ ] 不存在过拟合现象

## 6. 合规性规则

### 6.1 金融合规
- [ ] 模型符合金融行业监管要求
- [ ] 保留必要的审计追踪信息
- [ ] 符合数据隐私保护法规
- [ ] 支持模型解释性和可解释性

### 6.2 安全性
- [ ] 模型不会受到对抗性攻击
- [ ] 输入验证足够严格
- [ ] 模型输出具有可追溯性
- [ ] 防止模型窃取和滥用

## 7. 最佳实践规则

### 7.1 文档化
- [ ] 模型版本控制清晰
- [ ] 特征工程文档完整
- [ ] 模型假设说明清楚
- [ ] 验证过程记录完整

### 7.2 可重复性
- [ ] 实验可复现
- [ ] 结果一致性好
- [ ] 参数设置透明
- [ ] 环境依赖明确
