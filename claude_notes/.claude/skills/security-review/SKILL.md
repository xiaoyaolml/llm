# security-review

## 目标

识别会直接影响系统安全边界、数据保护和审计可信度的高风险问题。

## 适用条件

- 改动涉及认证、授权、输入处理、会话、密钥、日志、配置或依赖
- 需要做上线前安全审查
- 需要明确 blocking 级安全问题

## 审查重点

- 注入、越权、路径穿越、命令执行、敏感信息泄露
- 认证流程、令牌管理、权限边界和会话安全
- 密钥、加密、日志脱敏与审计链路
- 依赖组件、默认配置和错误信息暴露

## 输入期望

- 目标模块、接口或配置范围
- 安全标准、敏感数据类型、威胁模型
- 若有漏洞背景、事故线索或审计要求，可一并提供

## 输出要求

- 按 critical / high / medium / low 分级
- 指出位置、风险、利用方式和修复方向
- 说明是否阻塞上线
- 给出最小修复集与验证建议

## 协作边界

- 合规落地与审计控制转 `compliance-review`
- 协议暴露面和错误语义转 `protocol-validate`
