## 📊 验证维度

### 1. 数据完整性 (Data Completeness)
- **缺失值检测**: 是否存在缺失数据或空值
- **数据覆盖率**: 数据覆盖范围是否满足要求
- **连续性检查**: 数据序列是否连续，有无断点
- **字段完整性**: 所有必需字段是否都有值
- **时间序列完整性**: 时间戳是否连续且无重复

### 2. 数据准确性 (Data Accuracy)
- **价格验证**: 价格是否在合理范围内
- **量价关系**: 成交量与价格变动是否合理
- **涨跌幅限制**: 是否超过涨跌幅限制
- **除权除息处理**: 公司行为是否正确处理
- **货币单位**: 是否正确处理不同货币单位

### 3. 数据一致性 (Data Consistency)
- **跨市场一致性**: 相同标的在不同市场数据是否一致
- **数据源一致性**: 不同数据源之间的数据一致性
- **时间戳一致性**: 不同时区的时间戳处理是否一致
- **字段格式一致性**: 数据格式是否统一
- **参照数据一致性**: 标的名称、代码是否与参照数据一致

### 4. 数据时效性 (Data Timeliness)
- **实时延迟**: 实时数据延迟是否在可接受范围
- **更新频率**: 数据更新频率是否满足要求
- **时序正确性**: 数据时间戳是否正确排序
- **市场状态**: 是否正确处理开盘、收盘状态
- **时区处理**: 是否正确处理不同时区数据

### 5. 数据合规性 (Data Compliance)
- **数据许可**: 是否遵守数据许可协议
- **版权保护**: 是否正确标注数据版权信息
- **访问控制**: 是否正确控制数据访问权限
- **隐私保护**: 是否保护个人隐私信息
- **监管要求**: 是否满足金融监管数据要求

## 🛠️ 扩展方式

### 1. 添加新数据类型
**场景**: 你需要支持加密货币数据验证

**步骤**:
1. 编辑 `SKILL.md`，添加「加密货币数据」章节：
```markdown
#### 6. 加密货币数据验证
- [ ] **价格验证**: 价格是否在合理范围内（考虑加密货币高波动性）
- [ ] **流动性检查**: 交易对流动性是否充足
- [ ] **多交易所一致性**: 相同交易对在不同交易所的价格差异
- [ ] **链上数据验证**: 链上数据是否与中心化数据一致
- [ ] **DeFi数据验证**: DeFi协议数据准确性
```

2. 在 `CHECKLISTS/data_quality_checklists.md` 添加示例：
```markdown
## 加密货币数据检查清单

### 必查项
- [ ] 价格范围验证（考虑百倍涨幅可能性）
- [ ] 交易量是否非负且在合理范围
- [ ] 不同交易所价格差异验证
- [ ] 链上数据与中心化数据一致性
- [ ] DeFi协议数据准确性验证

### 高风险模式
| 模式 | 风险 | 修复建议 |
|------|------|----------|
| 未验证价格范围 | 价格异常影响策略 | 根据加密货币特点调整验证范围 |
| 缺少多交易所对比 | 数据可能被操控 | 实施跨交易所价格验证 |
| 未验证链上数据 | 数据准确性问题 | 实施链上数据验证机制 |
```
