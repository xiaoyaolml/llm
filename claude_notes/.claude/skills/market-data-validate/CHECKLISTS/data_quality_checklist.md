# 行情数据验证清单

## 使用时机

修改行情接入、数据清洗、快照合成、多源对齐、数据分发逻辑时使用。

## 完整性

- [ ] 缺失检测：序列号跳变、时间间隔异常、品种遗漏
- [ ] 重复检测：同一 seq/timestamp 重复到达
- [ ] 乱序检测：序列号或时间戳非单调递增
- [ ] 开盘/收盘/停牌状态变更正确处理
- [ ] 分时段数据（盘前/盘中/盘后）边界正确

## 准确性

- [ ] 价格跳变阈值（如涨跌停百分比）已配置并生效
- [ ] 精度一致：小数位数、最小变动单位与交易所定义吻合
- [ ] 汇率 / 币种基准正确
- [ ] 成交量 / 持仓量不出现负值
- [ ] 合约乘数 / 面值等静态字段与基准一致

## 时效性

- [ ] 延迟阈值明确（从交易所→本地的端到端）
- [ ] 过期数据有标记或丢弃策略
- [ ] 时钟偏差容忍范围已配置
- [ ] 网络抖动、重连后恢复逻辑正确

## 一致性

- [ ] 多源数据（主备/多交易所）对齐策略明确
- [ ] 快照 + 增量 = 当前状态 公式可验证
- [ ] 跨品种价差 / 关联价格无逻辑冲突
- [ ] 历史回放与实时路径使用相同清洗逻辑

## 异常处理

- [ ] 闪崩 / 熔断 / 停牌期间行为明确
- [ ] 非交易时段数据正确过滤或标记
- [ ] 数据源断连自动重连 + 状态恢复
- [ ] 异常事件有告警和日志

## 阻塞问题示例

- 快照与增量不一致导致价格错误
- 缺失检测失效导致空行情
- 延迟阈值无效导致使用过期数据
- 精度不一致导致计算偏差
