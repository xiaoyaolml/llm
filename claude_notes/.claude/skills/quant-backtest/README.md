## 📊 验证维度

### 1. 策略逻辑正确性
- **信号生成**: 买卖信号是否逻辑正确
- **仓位管理**: 仓位调整是否符合策略规则
- **风控机制**: 止损、止盈、最大持仓等是否生效
- **交易成本**: 手续费、滑点是否正确计算
- **时间处理**: 时区、交易时间是否正确处理

### 2. 数据质量验证
- **数据完整性**: 是否存在缺失数据或异常值
- **数据清洗**: 是否正确处理分红、拆股等公司行为
- **前窥偏差**: 是否避免使用未来数据
- **生存者偏差**: 是否包含已退市股票
- **数据频率**: 数据频率是否与策略匹配

### 3. 绩效指标计算
- **收益率**: 年化收益率、累计收益率计算正确
- **风险指标**: 夏普比率、最大回撤、波动率计算正确
- **交易统计**: 胜率、盈亏比、交易频率统计正确
- **基准比较**: 是否与基准指数进行合理比较
- **风险调整**: 是否考虑交易成本后的净收益

### 4. 过拟合检测
- **样本外测试**: 是否有独立的样本外测试
- **参数敏感性**: 策略参数是否过于敏感
- **数据挖掘偏差**: 是否过度优化历史数据
- **滚动回测**: 是否使用滚动窗口验证稳定性
- **交叉验证**: 是否使用多种市场环境验证

### 5. 回测框架验证
- **订单执行**: 订单是否按规则执行
- **撮合机制**: 撮合逻辑是否符合实际市场
- **资金管理**: 现金流、保证金是否正确处理
- **复权处理**: 价格是否正确复权
- **税务处理**: 是否考虑税收影响

## 🛠️ 扩展方式

### 1. 添加新策略类型
**场景**: 你需要支持强化学习策略验证

**步骤**:
1. 编辑 `SKILL.md`，添加「强化学习策略」章节：
```markdown
#### 6. 强化学习策略验证
- [ ] **环境设计**: 奖励函数是否合理设计
- [ ] **动作空间**: 动作定义是否符合交易规则
- [ ] **状态空间**: 状态表示是否包含足够信息
- [ ] **训练过程**: 是否避免过拟合和数据泄露
- [ ] **策略评估**: 是否有充分的离线和在线评估
```

2. 在 `CHECKLISTS/strategy_checklists.md` 添加示例：
```markdown
## 强化学习策略检查清单

### 必查项
- [ ] 奖励函数设计是否避免短期优化
- [ ] 动作空间是否包含持有现金选项
- [ ] 状态空间是否包含风险指标
- [ ] 训练数据是否与测试数据独立
- [ ] 是否使用经验回放避免相关性

### 高风险模式
| 模式 | 风险 | 修复建议 |
|------|------|----------|
| 过度优化奖励函数 | 策略泛化能力差 | 使用多个奖励函数测试鲁棒性 |
| 忽略交易成本 | 实盘表现差 | 在奖励函数中包含成本 |
| 数据泄露 | 过拟合 | 确保训练和测试数据完全独立 |
```
