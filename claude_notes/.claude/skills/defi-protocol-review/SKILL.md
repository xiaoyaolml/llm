# defi-protocol-review

## 目标

识别会影响协议资金安全、权限边界、价格可信度和升级可控性的关键问题。

## 适用条件

- 涉及智能合约协议、流动性池、清算、治理、预言机或跨链逻辑
- 需要做上线前协议复核或审计辅助
- 需要从资金和状态机角度审查设计

## 审查重点

- 资产流转、状态变更和不变量
- 权限、治理、升级、暂停和恢复机制
- 预言机、精度、价格操纵和流动性风险
- 事件、审计、监控和异常恢复能力

## 输入期望

- 协议角色、核心流程、风险边界
- 关键合约、状态变量、事件和权限模型
- 若有已知攻击面或审计结论，可一并提供

## 输出要求

- 先给资金与状态机风险概览
- 再给权限、价格、流动性和升级层建议
- 标出直接阻塞上线的高危问题
- 给出验证、演练和监控建议

## 协作边界

- 通用漏洞与审计问题转 `security-review`
- 接口契约问题转 `api-design-review`
