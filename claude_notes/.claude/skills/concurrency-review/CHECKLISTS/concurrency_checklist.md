# 并发审查清单

## 使用时机

修改 mutex / atomic / 线程池 / 队列 / 回调注册 / 连接管理 / shutdown 路径时使用。

## 线程安全

- [ ] 共享数据的所有读写点已识别
- [ ] 每个共享字段的保护方式已明确（mutex / atomic / 无共享）
- [ ] 锁粒度合理：无过粗（阻塞无关线程）也无过细（增加遗漏风险）
- [ ] 锁获取顺序全局一致，无循环依赖
- [ ] 无条件变量虚假唤醒遗漏（while 而非 if）
- [ ] 无 TOCTOU（check-then-act 跨越锁边界）

## 生命周期

- [ ] 异步回调不持有裸指针（优先 weak_ptr / shared_ptr）
- [ ] 连接关闭 / 超时 / 取消后不继续访问失效资源
- [ ] 对象销毁顺序明确，析构不依赖已销毁的成员
- [ ] shutdown 路径能在有限时间内收敛所有后台任务
- [ ] 定时器 / 重试回调在目标对象销毁后不触发

## 原子与无锁

- [ ] atomic 变量用途和保护范围明确
- [ ] memory_order 选择有注释说明（非默认 seq_cst 时）
- [ ] 无锁结构的 ABA 问题已识别并防护
- [ ] CAS 循环有退避策略或最大重试次数
- [ ] 不存在混用锁和无锁路径导致的不一致

## 阻塞问题示例（直接阻塞合并）

- 明确存在数据竞争且无保护
- shutdown 死锁或无法收敛
- 回调持有裸指针且目标生命周期无保障
- memory_order_relaxed 用于需要 acquire-release 的场景
