## 简介

**Claude Code** 是一个运行在终端/集成开发环境IDE 中的 AI 编程助手，可以直接读取、编辑文件，运行命令，管理 git 等，核心能力包括：

- 📖 阅读和理解代码
- ✏️ 编辑和创建文件
- 🔍 搜索和探索项目
- 🚀 开发和调试程序
- 📤 管理版本控制
- 🛠️ 自动化开发工作流

## 核心操作

### 1. 对话

直接在终端输入自然语言指令，例如：

* "给这个函数添加错误处理"
* "解释一下这个文件在做什么"
* "修复这个 bug"

### 2. 常用斜杠命令

| 命令         | 作用                   |
| ------------ | ---------------------- |
| `/help`    | 查看帮助               |
| `/clear`   | 清空对话历史           |
| `/fast`    | 切换快速模式           |
| `/compact` | 压缩上下文以节省 token |

### 3. 权限模式

Claude Code 运行时有不同的权限级别：

* **自动模式** ：某些操作（如读取文件）自动执行
* **确认模式** ：写文件、运行命令等需要你确认

你会看到工具调用提示，选择 **Approve** 或  **Deny** 。

### 4. 核心能力

* 读取文件 — 直接查看代码
* 编辑文件 — 精确修改代码（Edit 工具）
* 运行命令 — 执行 shell 命令
* 搜索 — 用 Grep/Glob 快速定位代码
* Git 管理 — 查看状态、创建 commit、PR 等

## 核心功能

### 1. **文件阅读** - Read 工具

用于快速查看文件内容：

```
最常用场景：
- 理解现有代码结构
- 查看配置文件
- 快速浏览长文件
```

**使用示例：**

```
"Read /path/to/file.py"
→ 显示从第1行开始的内容
```

### 2. **文件搜索** - Glob 工具

快速找到文件：

```
常见模式：
- **/*.py       # 所有 Python 文件
- src/**/*.ts   # src 目录下所有 TypeScript
- test_*.py     # 以 test_ 开头的文件
```

### 3. **内容搜索** - Grep 工具

在文件中搜索特定文本：

```
功能：
- 正则表达式搜索
- 范围搜索（-B/-A/-C）
- 按文件类型过滤
- 计数和列表模式
```

## 文件操作

### 📝 三种编辑方式

| 工具                   | 用途               | 场景               |
| ---------------------- | ------------------ | ------------------ |
| **Edit**         | 精确替换字符串     | 修改现有代码       |
| **Write**        | 创建或完全覆盖     | 新建文件或完全重写 |
| **NotebookEdit** | 编辑 Jupyter cells | 数据科学项目       |

#### Edit 工具 - 精确替换

```
优点：
✓ 低错误率 - 需要唯一的 old_string
✓ 版本控制友好 - 只显示变化部分
✓ 安全 - 必须先读文件

使用流程：
1. 先用 Read 读取源文件
2. 复制要替换的确切文本
3. 使用 Edit，指定 old_string 和 new_string
```

#### Write 工具 - 文件创建

```
使用场景：
- 创建全新文件
- 完全重写文件内容
- 不涉及现有数据的场景

注意：会覆盖已有文件！
```

## 代码开发

### 1. **代码探索** - Agent 工具

用于复杂的代码理解任务：

```
子代理类型：
┌─────────────┬──────────────────────┐
│ Explore     │ 快速探索代码库       │
│ Plan        │ 设计实现方案         │
│ general     │ 通用研究和多步骤任务 │
└─────────────┴──────────────────────┘
```

**何时使用 Explore 代理：**

- 需要查找多个文件
- 理解复杂的代码模式
- 确定最佳实现方式

### 2. **规划任务** - EnterPlanMode

在开始编码前设计方案：

```
适合场景：
✓ 新功能实现
✓ 代码重构
✓ 架构设计
✓ 不确定如何开始

流程：
1. 设置 EnterPlanMode
2. 探索现有代码
3. AskUserQuestion 确认方向
4. ExitPlanMode 保存计划
5. 开始实现
```

### 3. **跟踪进度** - TodoWrite 工具

组织和管理任务：

```
任务状态：
- pending      # 未开始
- in_progress  # 进行中
- completed    # 已完成

最佳实践：
✓ 多步骤任务时使用
✓ 完成立即标记，不要批量处理
✓ 同时只有一个 in_progress 任务
```

## Git 工作流

### 基础 Git 操作

```bash
# 查看状态
git status

# 查看提交记录
git log --oneline

# 比较差异
git diff

# 创建分支
git branch [branch-name]

# 切换分支
git checkout [branch-name]

# 提交代码
git commit -m "message"

# 推送到远程
git push
```

### 使用 Claude Code 的 Git 工具

| 工具                                  | 功能           |
| ------------------------------------- | -------------- |
| `mcp__gitkraken__git_status`        | 查看工作区状态 |
| `mcp__gitkraken__git_add_or_commit` | 暂存和提交     |
| `mcp__gitkraken__git_log_or_diff`   | 查看历史和变化 |
| `mcp__gitkraken__git_branch`        | 管理分支       |
| `mcp__gitkraken__git_push`          | 推送代码       |

## 高效使用技巧

* 具体描述需求 — "把 login.py 第 20 行的错误提示改成中文" 比 "改一下" 更好
* 分步骤给复杂任务 — 大任务拆解成小步骤，逐步确认
* 拒绝不需要的改动 — 如果 AI 改多了，可以直接拒绝并要求缩小范围
* 引用文件/行号 — 指向具体位置可以更精准地操作

## 高级功能

### 1. **持久内存** - 自动记忆系统

跨会话保存信息：

```
内存文件位置：
~/.claude/projects/[project-name]/memory/

保存内容：
✓ 稳定的模式和习惯
✓ 项目结构和架构
✓ 用户偏好设置
✓ 解决方案和调试技巧

不保存：
✗ 临时会话信息
✗ 未验证的内容
✗ 重复信息
```

### 2. **后台任务** - run_in_background

异步执行长时间操作：

```
使用场景：
- 构建和编译
- 测试执行
- 数据处理
- 文件转换

用法：
agent(..., run_in_background=true)
→ 任务继续进行，您可以做其他工作
```

### 3. **工作树** - Worktree

隔离的开发环境：

```
优点：
✓ 同时处理多个分支
✓ 隔离更改
✓ 清晰的工作边界

使用：
EnterWorktree → 工作 → ExitWorktree
```

### 4. **定时任务** - CronCreate

设置重复执行的任务：

```
语法：标准 5 字段 cron 表达式
示例：
- "*/5 * * * *"    # 每 5 分钟
- "0 9 * * *"      # 每天早上 9 点
- "0 9 * * 1-5"    # 工作日 9 点
```

## 最佳实践

### ✅ 代码审查和质量

```
1. 读代码前先理解需求
   → Read 查看现有实现
   → Grep 搜索相关代码

2. 编辑时低风险操作
   → 优先使用 Edit（不是 Write）
   → 使用 -B/-A 参数显示上下文

3. 测试和验证
   → 改完后立即运行测试
   → 确保没有引入回归
```

### 🎯 工作流优化

```
快速任务（修复 2-3 行）:
Read → Edit → (可选) Commit

中等任务（多文件改动）:
Glob/Grep → Read → Edit → TodoWrite → Test → Commit

复杂任务（新功能）:
EnterPlanMode → Explore → Plan → AskUserQuestion 
→ ExitPlanMode → Agent(implement) → Test → PR

数据分析/大型查询:
Agent(Explore,级别=very_thorough) → 并行多个查询
```

### 🔒 安全事项

```
需要确认再执行的操作：
⚠️ git reset --hard
⚠️ git push --force
⚠️ 删除文件/分支
⚠️ 修改共享配置

自动可执行的操作：
✓ 创建本地分支
✓ 编辑文件
✓ 运行测试
✓ 提交本地变更
```

### 📊 参数和选项

```
并行执行：
- 用单个消息调用多个独立工具
- 加快处理速度
- 提高效率

依赖关系：
- 必须等待前一个操作完成
- 按顺序执行
- 避免猜测后续参数
```

## 💡 常见问题解答

**Q: 什么时候用 Agent vs 直接使用 Grep?**

- 简单搜索 → 用 Grep
- 需要多轮查询 → 用 Agent

**Q: Edit 失败说不唯一，怎么办?**

- 添加更多上下文
- 通过 `-B/-A` 参数显示更多行
- 查看文件验证确切内容

**Q: 如何安全地处理大文件?**

- 使用 offset/limit 参数
- 分段读取（如 PDF 按页读）
- 使用 head_limit 限制输出

**Q: 什么时候需要创建 Worktree?**

- 需要同时处理多个分支
- 长期独立工作
- 避免搅乱主工作区
