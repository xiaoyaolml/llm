# protocol-validate

## 目标

识别协议实现中的兼容性问题、边界条件缺失、错误语义不稳定和会话处理缺陷。

## 适用条件

- 目标代码包含编码器、解码器、协议网关、会话管理或报文定义
- 改动涉及字段增删、版本升级、错误码或重传逻辑
- 需要专项验证协议正确性和兼容性

## 审查重点

- 报文边界、字段顺序、类型、长度和校验
- 半包、坏包、乱序、重复、超时和重放处理
- 会话建立、心跳、序列号、重传和关闭流程
- 向后兼容、版本演进和错误语义稳定性

## 输入期望

- 协议类型、版本、目标模块和调用方背景
- 若有协议文档、样例报文或兼容性要求，建议提供
- 若有线上错误、抓包或异常日志，可一起提供

## 输出要求

- 先给兼容性和正确性风险概览
- 再给字段级、会话级、错误处理级问题
- 区分 blocking 问题与实现优化问题
- 给出测试建议，如坏包、半包、升级兼容回归

## 协作边界

- 网络延迟与传输效率问题转 `network-optimization`
- API 合同层问题转 `api-design-review`
