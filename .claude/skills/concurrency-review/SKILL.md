# concurrency-review

## 目标

识别并发路径中的竞态、死锁、生命周期失配、背压失控和收敛失败等高风险问题。

## 适用条件

- 目标代码包含线程、锁、原子、回调、队列、线程池
- 问题表现为卡死、堆积、偶发崩溃、关闭不干净、尾延迟抖动
- 需要专项分析线程安全和并发行为

## 审查重点

- 共享状态读写点与同步策略
- 锁顺序、锁粒度、等待链路
- 原子变量用途与内存序假设
- 回调、定时器、异步任务的生命周期
- shutdown、close、cancel 的收敛行为

## 输入期望

- 相关模块、线程模型或调用链描述
- 具体现象，如死锁、偶发崩溃、延迟抖动、任务积压
- 若有日志、栈、压测现象，建议一起提供

## 输出要求

- 先列并发模型和关键共享状态
- 再列 blocking 风险和证据
- 指出 race / deadlock / ABA / use-after-free 等类别
- 给出最小风险修复顺序和验证建议
