# network-optimization

## 目标

识别网络链路中的往返放大、重试放大、连接不稳和数据传输低效问题，并给出可验证的优化方向。

## 适用条件

- 涉及客户端、网关、RPC、行情推送、连接池或传输协议
- 问题表现为网络尾延迟高、重传多、连接抖动或带宽浪费
- 需要专项分析网络路径成本

## 审查重点

- 连接建立、复用、断开与恢复策略
- 超时、重试、幂等性和背压边界
- 批处理、压缩、编解码、零拷贝与缓冲区管理
- 丢包、乱序、重复、排序和恢复策略

## 输入期望

- 网络角色、链路结构、协议类型
- 延迟或稳定性现象
- 若有抓包、指标、trace 或错误码统计，建议提供

## 输出要求

- 先描述网络路径与关键放大点
- 再列高风险问题和最小优化方案
- 说明对 tail latency、吞吐和兼容性的影响
- 给出验证建议，如压测、抓包或指标观察

## 协作边界

- 协议正确性和版本兼容性转 `protocol-validate`
- 通用性能热点定位转 `performance-bottleneck-identification`
