# 金融风险计算规则库

## 1. 数值稳定性规则

### 1.1 极端条件处理
- [ ] **价格为0**: 所有收益率计算必须处理价格为0的情况
- [ ] **负价格**: 商品期货可能出现负价格，算法必须支持
- [ ] **极大值**: 处理股票拆分、反向拆分导致的极大/极小价格
- [ ] **非交易日**: 周末、节假日的数据缺失处理

### 1.2 除零保护
- [ ] **收益率计算**: `(P1 - P0) / P0` 必须检查 `P0 == 0`
- [ ] **波动率计算**: 标准差分母必须检查样本数量 > 1
- [ ] **相关系数**: 协方差矩阵对角线元素必须非零
- [ ] **杠杆计算**: 资产负债比率必须检查分母非零

### 1.3 溢出保护
- [ ] **指数运算**: `exp(x)` 必须检查 `x` 的范围（避免 inf）
- [ ] **对数运算**: `log(x)` 必须检查 `x > 0`
- [ ] **幂运算**: `pow(x, y)` 必须检查底数和指数的合法性
- [ ] **累积乘积**: 长时间序列的累积乘积可能溢出

### 1.4 收敛性保证
- [ ] **迭代算法**: 必须设置最大迭代次数
- [ ] **数值积分**: 必须设置收敛容差和最大步数
- [ ] **优化算法**: 必须检查梯度范数和函数值变化
- [ ] **根查找**: 必须验证区间端点函数值符号相反

## 2. 精度控制规则

### 2.1 货币精度
- [ ] **财务计算**: 所有货币计算使用 `decimal` 或固定点数
- [ ] **避免浮点**: 禁止在财务计算中使用 `float`/`double`
- [ ] **舍入策略**: 统一使用银行家舍入法（四舍六入五成双）
- [ ] **精度保持**: 中间计算保持更高精度，最后舍入

### 2.2 浮点比较
- [ ] **相等比较**: 使用 `abs(a - b) < epsilon` 而非 `a == b`
- [ ] **大小比较**: 考虑浮点误差的边界情况
- [ ] **NaN 处理**: 明确处理 NaN 值的传播
- [ ] **无穷大处理**: 明确处理 inf 值的行为

### 2.3 累积误差
- [ ] **Kahan 求和**: 大量数字求和使用 Kahan 算法
- [ ] **中间精度**: 保持中间计算的高精度
- [ ] **误差分析**: 对关键计算进行误差传播分析
- [ ] **基准测试**: 与高精度参考实现比较

## 3. 蒙特卡洛模拟规则

### 3.1 随机数生成
- [ ] **RNG 质量**: 使用通过统计测试的 RNG（MT19937、PCG）
- [ ] **种子管理**: 使用硬件随机源初始化，避免重复
- [ ] **线程安全**: 每个线程独立 RNG 实例
- [ ] **可重现性**: 提供固定种子用于调试和测试

### 3.2 路径生成
- [ ] **低差异序列**: 使用 Sobol、Halton 序列提高收敛速度
- [ ] **维度处理**: 高维问题使用合适的路径生成方法
- [ ] **相关性**: 正确处理多资产的相关性结构
- [ ] **跳跃过程**: 包含跳跃扩散模型的支持

### 3.3 方差缩减
- [ ] **控制变量**: 使用已知解析解作为控制变量
- [ ] **重要性采样**: 对尾部事件使用重要性采样
- [ ] **分层抽样**: 对关键区域使用分层抽样
- [ ] **对偶变量**: 使用对偶变量减少方差

### 3.4 收敛验证
- [ ] **路径数量**: 足够的路径数量（>10000 用于 VaR）
- [ ] **置信区间**: 计算结果的置信区间
- [ ] **收敛图**: 绘制估计值随路径数的变化
- [ ] **统计测试**: 使用 Kolmogorov-Smirnov 测试分布拟合

## 4. 风险指标计算规则

### 4.1 VaR 计算
- [ ] **历史窗口**: 至少 250 个交易日的历史数据
- [ ] **危机数据**: 包含 2008 年金融危机等极端事件
- [ ] **分位数插值**: 使用线性或样条插值提高精度
- [ ] **回测验证**: 定期进行 VaR 回测验证

### 4.2 CVaR 计算
- [ ] **尾部平均**: 正确计算超出 VaR 的损失平均值
- [ ] **一致性**: CVaR 必须大于等于 VaR
- [ ] **稳定性**: 尾部样本足够多以保证稳定性
- [ ] **敏感性**: 测试不同置信水平的影响

### 4.3 Greeks 计算
- [ ] **解析解优先**: 优先使用解析解而非数值微分
- [ ] **步长选择**: 数值微分步长合理（1e-4 到 1e-6）
- [ ] **一致性检查**: Delta-Gamma 近似的一致性验证
- [ ] **曲面处理**: 考虑波动率曲面和利率曲面

### 4.4 压力测试
- [ ] **历史场景**: 包含历史极端事件（1987、2008、2020）
- [ ] **假设场景**: 包含合理的假设场景（利率冲击、汇率冲击）
- [ ] **组合效应**: 考虑多个风险因子同时冲击
- [ ] **流动性影响**: 考虑市场流动性枯竭的影响

## 5. 边界条件规则

### 5.1 数据边界
- [ ] **空数据集**: 无交易数据时的默认行为
- [ ] **单点数据**: 只有一个数据点时的统计处理
- [ ] **缺失数据**: 处理缺失数据的插值或删除策略
- [ ] **异常值**: 识别和处理异常值（Z-score > 3）

### 5.2 时间边界
- [ ] **非交易时间**: 处理盘前盘后、周末、节假日
- [ ] **时区处理**: 正确处理不同时区的市场数据
- [ ] **夏令时**: 处理夏令时切换的影响
- [ ] **闰秒**: 考虑闰秒对高频数据的影响

### 5.3 市场边界
- [ ] **熔断机制**: 处理市场熔断时的数据
- [ ] **涨跌停板**: 处理价格限制的影响
- [ ] **流动性枯竭**: 处理买卖价差极大的情况
- [ ] **市场关闭**: 处理交易所临时关闭的情况

## 6. 严重级别标准

### 高危 (必须修复)
- 导致错误风险评估的计算错误
- 未处理的除零或溢出错误
- 错误的随机数生成或种子管理
- 不正确的 VaR 或 Greeks 计算

### 中危 (建议修复)
- 次优的数值方法选择
- 未充分的压力测试覆盖
- 未优化的蒙特卡洛收敛
- 未验证的边界条件处理

### 低危 (可选修复)
- 未注释的复杂公式
- 未记录的假设条件
- 未优化的代码结构
- 未使用的调试代码

## 7. 风险计算测试标准

### 7.1 数值验证
- **精度要求**: 财务计算精度 ≥ 1e-6
- **稳定性要求**: 极端条件下不崩溃
- **一致性要求**: 相关指标逻辑一致
- **可重现性**: 相同输入产生相同输出

### 7.2 统计验证
- **VaR 回测**: 违约频率符合预期
- **分布拟合**: KS 测试 p-value > 0.05
- **收敛性**: 蒙特卡洛估计随路径数收敛
- **敏感性**: 参数变化产生合理响应

### 7.3 业务验证
- **监管合规**: 符合 Basel III、FRTB 等要求
- **风险管理**: 能够识别极端损失
- **对冲有效性**: Greeks 计算支持有效对冲
- **报告准确性**: 监管报告数据准确

## 8. 风险计算反模式

### 8.1 常见错误
- ❌ 使用 `float` 进行货币计算
- ❌ 忽略极端市场价格条件
- ❌ 固定随机数种子用于生产环境
- ❌ 未验证数值方法的收敛性

### 8.2 金融特定错误
- ❌ 使用正态分布假设极端尾部
- ❌ 忽略资产间的相关性结构
- ❌ 未包含历史危机数据
- ❌ 未进行 VaR 回测验证
