# risk-check

## 目标

识别会导致风险结果偏差、边界失真、数值不稳定或口径不一致的问题。

## 适用条件

- 涉及 VaR、Greeks、蒙特卡洛、压力测试、限额、估值或相关性计算
- 需要验证风险模型改动或结果异常
- 需要从数值和金融语义双重角度审查

## 审查重点

- 精度、舍入、溢出、下溢和收敛性
- 输入数据质量、极端场景和假设边界
- 风险指标定义、窗口、采样与口径一致性
- 回归基线、对账和解释性

## 输入期望

- 风险类型、模型背景、目标模块
- 样例输入、预期结果、历史基线或偏差现象
- 若有监管或业务约束，建议一起提供

## 输出要求

- 先给数值和语义风险概览
- 再给边界条件、模型假设、结果验证建议
- 标出可能影响生产决策的 blocking 问题
- 给出回归测试、对账或抽样验证方案

## 协作边界

- 数据质量问题转 `market-data-validate`
- API 或报表口径暴露问题可联动 `api-design-review`
