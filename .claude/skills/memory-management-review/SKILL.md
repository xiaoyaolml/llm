# memory-management-review

## 目标

识别 C++ 代码中的所有权不清、资源释放不完整、泄漏、悬垂引用和高频分配热点。

## 适用条件

- 涉及裸指针、智能指针、对象池、分配器、缓存结构
- 涉及 shutdown、异常路径、资源释放、缓存淘汰
- 需要专项排查内存安全与分配成本

## 审查重点

- unique/shared/weak/raw 指针使用是否合理
- RAII、异常安全和析构路径是否完整
- 分配频率、复用策略、对象池参数是否合理
- 悬垂引用、双重释放、越界、碎片化风险
- 对齐、缓存局部性和回收成本

## 输入期望

- 相关类、路径或内存热点描述
- 是否已有泄漏现象、OOM、抖动或长时间增长问题
- 若有 profiler、sanitizer 或 crash 线索，建议一并提供

## 输出要求

- 区分安全问题与性能问题
- 标明所有权模型和高风险释放路径
- 给出最小修改建议与验证方式
- 推荐 sanitizer、回归测试或压测验证点

## 协作边界

- 并发回收与 lock-free 回收问题转 `concurrency-review`
- 服务端整体设计问题转 `cpp-backend-review`
