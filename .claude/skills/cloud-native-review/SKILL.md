# cloud-native-review

## 目标

审查系统在容器化和平台化环境中的部署质量、恢复能力、配置安全与运维可交付性。

## 适用条件

- 涉及 Docker、Kubernetes、配置中心、服务发现、资源限制
- 准备上线、迁移、扩容或做平台治理
- 需要从交付和运行层面审查系统

## 审查重点

- 镜像、配置、密钥、权限与默认值
- 健康检查、超时、重试、回滚和故障恢复
- 弹性、资源配额、容量与依赖管理
- 与指标、日志、trace、告警的联动

## 输入期望

- 部署形态、运行环境、资源目标
- 配置来源、依赖清单、健康检查策略
- 若有事故背景或上线窗口，建议说明

## 输出要求

- 先给交付与运行风险概览
- 再给配置、安全、弹性、恢复等维度建议
- 标出上线前必须补齐项
- 给出最小运行基线和验证建议

## 协作边界

- 监控与 runbook 细节转 `system-monitoring`
- 认证、授权、密钥等问题转 `security-review`
