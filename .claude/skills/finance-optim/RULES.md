# 金融C++性能优化规则库

## 1. 延迟优化规则

### 1.1 内存分配优化
- [ ] **高频路径禁止动态分配**：交易引擎核心循环不得使用 `new`/`malloc`
- [ ] **对象池模式**：预分配高频对象，使用 `thread_local` 池
- [ ] **栈分配**：临时对象优先使用栈分配（<1KB）
- [ ] **自定义分配器**：容器使用内存池分配器（`std::pmr::memory_resource`）

### 1.2 函数调用优化
- [ ] **避免虚函数**：高频路径使用 CRTP（奇异递归模板模式）或模板
- [ ] **内联小函数**：标记为 `inline` 或 `__attribute__((always_inline))`
- [ ] **尾递归优化**：编译器支持时使用尾递归
- [ ] **函数指针**：避免间接调用，使用直接调用或模板

### 1.3 并发优化
- [ ] **无锁数据结构**：使用 `std::atomic` + CAS 操作
- [ ] **读写锁**：读多写少场景使用 `std::shared_mutex`
- [ ] **线程亲和性**：关键线程绑定到特定 CPU 核心
- [ ] **NUMA 优化**：内存分配在本地 NUMA 节点

## 2. 金融特定优化规则

### 2.1 货币计算
- [ ] **整数运算**：所有货币计算使用 `int64_t`（单位：分）
- [ ] **避免浮点**：禁止在财务计算中使用 `float`/`double`
- [ ] **四舍五入**：统一四舍五入规则（银行家舍入法）
- [ ] **溢出检查**：关键计算检查整数溢出

### 2.2 价格处理
- [ ] **价格档位**：使用 `std::unordered_map` 而非 `std::map`
- [ ] **快速查找**：二分查找或哈希表优化
- [ ] **批量更新**：市场数据批量处理，避免单条更新
- [ ] **增量计算**：只重新计算变更部分

### 2.3 随机数生成
- [ ] **快速 RNG**：蒙特卡洛使用 XorShift128+ 而非 MT19937
- [ ] **线程安全**：每个线程独立 RNG 实例
- [ ] **种子管理**：使用硬件随机源初始化
- [ ] **向量化生成**：批量生成随机数

## 3. 内存效率规则

### 3.1 数据结构优化
- [ ] **连续内存**：使用 `std::vector` 而非 `std::list`
- [ ] **缓存行对齐**：关键数据结构 `alignas(64)`
- [ ] **避免 false sharing**：添加 padding 字节
- [ ] **位域压缩**：布尔标志使用位域

### 3.2 零拷贝技术
- [ ] **引用传递**：大型对象使用 `const&` 传递
- [ ] **移动语义**：临时对象使用 `std::move`
- [ ] **共享内存**：进程间通信使用共享内存
- [ ] **内存映射**：大文件使用 `mmap`

## 4. 编译器优化规则

### 4.1 编译选项
- [ ] **优化级别**：生产环境 `-O3 -march=native`
- [ ] **LTO**：启用链接时优化 `-flto`
- [ ] **Profile-guided**：使用 PGO 优化热点代码
- [ ] **安全检查**：生产环境移除断言 `-DNDEBUG`

### 4.2 代码提示
- [ ] **分支预测**：`__builtin_expect(condition, 1)`
- [ ] **循环展开**：手动展开关键循环
- [ ] **缓存预取**：`__builtin_prefetch`
- [ ] **内存屏障**：正确使用 `std::atomic_thread_fence`

## 5. 金融系统特定规则

### 5.1 交易引擎
- [ ] **订单匹配**：O(1) 时间复杂度查找
- [ ] **价格更新**：无锁更新价格档位
- [ ] **风险检查**：流式风险计算，避免阻塞
- [ ] **网络 I/O**：使用 DPDK 或 io_uring

### 5.2 风险计算
- [ ] **并行化**：OpenMP 或 TBB 并行计算
- [ ] **SIMD 优化**：AVX2/AVX512 向量化
- [ ] **增量计算**：只重新计算变更部分
- [ ] **预计算**：缓存常用计算结果

### 5.3 市场数据
- [ ] **压缩存储**：tick 数据使用 delta 编码
- [ ] **流式处理**：避免全量加载
- [ ] **预热缓存**：启动时预热关键数据
- [ ] **批量处理**：合并微小更新

## 6. 严重级别标准

### 高危 (必须修复)
- 每次执行 >1μs 开销的操作
- 动态内存分配在 >1KHz 路径
- 虚函数调用在 >10KHz 路径
- 锁竞争在高频路径

### 中危 (建议修复)
- 未对齐内存访问
- 未优化的数据结构
- 未充分利用硬件特性
- 次优算法选择

### 低危 (可选修复)
- 未使用 `const` 修饰符
- 未优化的循环变量
- 未使用的编译器提示
- 代码风格问题

## 7. 金融性能测试标准

### 7.1 延迟测试
- **P99 延迟**：<100μs (高频交易)
- **P999 延迟**：<1ms (风险计算)
- **尾部延迟**：<10ms (报告生成)

### 7.2 吞吐量测试
- **订单处理**：>100K TPS
- **风险计算**：>10K 计算/秒
- **数据处理**：>1GB/s

### 7.3 内存测试
- **GC 压力**：0 (无垃圾回收)
- **内存碎片**：<5%
- **缓存命中率**：>95% (L1), >85% (L2)

## 8. 优化反模式

### 8.1 常见错误
- ❌ 在高频路径使用 `std::cout`/日志
- ❌ 未预热的 JIT 代码
- ❌ 未对齐的 SIMD 操作
- ❌ 未处理的整数溢出

### 8.2 金融特定错误
- ❌ 使用 `double` 计算货币金额
- ❌ 未处理的市场数据乱序
- ❌ 未同步的时钟源
- ❌ 未验证的随机数质量
