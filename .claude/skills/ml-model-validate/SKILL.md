# ml-model-validate

## 目标

识别会导致模型离线结果和线上表现偏离的关键问题，并给出验证与监控建议。

## 适用条件

- 涉及训练、评估、推理、特征工程或模型上线
- 模型效果波动、漂移、偏差或线上表现下降
- 需要专项验证模型链路

## 审查重点

- 训练/验证/推理数据与流程是否一致
- 特征泄漏、标签污染、采样偏差和样本漂移
- 指标选择、阈值设定、稳定性和解释性
- 线上监控、回滚和再训练触发条件

## 输入期望

- 模型类型、任务目标、关键指标
- 数据和特征来源、训练与推理流程
- 若有线上异常或回归样本，建议提供

## 输出要求

- 先给验证风险概览
- 再给数据、特征、指标和线上监控问题
- 标出会导致上线失败或误判的 blocking 问题
- 给出补实验、补监控和回归建议

## 协作边界

- 金融风险语义问题可联动 `risk-check`
- 线上观测和告警问题可联动 `system-monitoring`
