# 行情数据验证示例

## 示例输入

```text
请参考 market-data-validate 的审查清单，审查以下行情数据处理逻辑：
（粘贴行情接收/清洗/分发代码）
```

## 期望输出

### 问题 1（阻塞）

- **位置**：`MarketDataHandler::onSnapshot()` 第 45 行
- **类型**：一致性
- **风险**：快照序列号与增量序列号不连续时，未触发全量重建，导致 orderbook 状态不一致
- **修复**：检测 gap 后请求全量快照，丢弃该 gap 之前的增量
- **验证**：构造序列号跳变的测试用例

### 问题 2（阻塞）

- **位置**：`PriceFilter::check()` 第 78 行
- **类型**：准确性
- **风险**：涨跌停百分比阈值硬编码为 10%，不适用于 ST 股票（5%）和科创板（20%）
- **修复**：从合约静态数据表获取对应涨跌幅限制
- **验证**：多品种测试覆盖

### 问题 3（非阻塞）

- **位置**：`LatencyMonitor::record()` 第 112 行
- **类型**：时效性
- **建议**：当前仅记录平均延迟，建议增加 P99/P999 分位数
