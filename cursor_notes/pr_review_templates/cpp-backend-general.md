## 2. 建议先发给 Cursor 的 Prompt

```text
先不要修改代码。
请作为 C++ 服务端 PR 审查者，审查当前改动，并按以下结构输出：
1. 正确性与边界条件
2. 并发与生命周期风险
3. 性能与尾延迟风险
4. 协议 / API / 配置兼容性影响
5. 日志、监控、trace 是否足够
6. 测试缺口
7. 必须阻塞合并的问题
```

## 3. Reviewer Checklist

### 正确性

- [ ] 空输入、空集合、空指针、错误状态已考虑
- [ ] 边界值、极值、非法输入已考虑
- [ ] 错误路径不会泄漏资源
- [ ] 返回值 / 错误码语义清晰

### 工程一致性

- [ ] 复用了现有模式，没有平地起新抽象
- [ ] 命名、分层、日志风格与现有代码一致
- [ ] 改动范围与目标一致，没有大面积顺手重构

### 可交付性

- [ ] 单元测试 / 回归测试明确
- [ ] 需要的文档更新已说明
- [ ] 风险和回滚方式可描述
