# C++ 服务端 PR 审查模板：安全 / 协议 / 风险

## 1. 适用场景

- 身份认证、授权、配置、安全审计改动
- 协议结构、消息字段、API 返回结构改动
- 风险计算、市场数据、回测或模拟逻辑改动

## 2. 建议先发给 Cursor 的 Prompt

```text
先不要修改代码。
请从安全、协议、风险和数据角度审查当前 PR：
1. 输入验证与注入风险
2. 授权边界与审计影响
3. 协议 / API 兼容性影响
4. 数值精度、单位、边界条件
5. 数据完整性、一致性、时效性
6. 回归测试与上线风险
```

## 3. Reviewer Checklist

### 安全

- [ ] 无明显注入、越权、敏感信息泄露风险
- [ ] 日志未打印秘密、token、敏感载荷
- [ ] 配置默认值安全
- [ ] 审计链路未被削弱

### 协议 / API

- [ ] 兼容性影响已说明
- [ ] 版本升级路径已说明
- [ ] 错误码与返回语义稳定
- [ ] 半包、坏包、非法输入路径已考虑

### 风险 / 数据

- [ ] rounding / overflow / underflow 已考虑
- [ ] 价格、数量、时间戳单位明确
- [ ] 数据去重、乱序、延迟处理明确
- [ ] 回归测试覆盖关键边界

## 4. 结论建议

### Blocking

- 存在安全高危风险
- 协议兼容性被破坏且无迁移方案
- 数值逻辑可能导致错误结果

### Non-blocking

- 审计、指标、文档仍可补强
- 建议增加数据异常样例测试
