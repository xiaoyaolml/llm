# C++ 服务端 PR 审查模板：性能 / 低延迟

## 1. 适用场景

- 热路径优化
- 网络 / RPC / 序列化路径变更
- 内存分配策略变更
- 日志、批处理、队列、缓存策略变更

## 2. 建议先发给 Cursor 的 Prompt

```text
先不要修改代码。
请重点从性能和低延迟角度审查当前 PR：
1. 热路径和额外工作量
2. 分配、拷贝、格式化开销
3. 锁争用、队列积压、背压
4. 网络 / I/O / 数据库放大效应
5. P99/P999 风险
6. benchmark / profiling 验证建议
7. 是否值得阻塞合并
```

## 3. Reviewer Checklist

### 热路径

- [ ] 是否把额外逻辑放进了热路径
- [ ] 是否新增不必要拷贝 / 临时字符串 / 动态分配
- [ ] 是否在锁内做了重活

### 延迟风险

- [ ] 是否可能放大 tail latency
- [ ] 是否引入额外同步点
- [ ] 是否改变批处理 / flush / retry 行为
- [ ] 是否影响背压与队列深度

### 证据与验证

- [ ] 是否有 benchmark / profiling 依据
- [ ] 是否说明目标预算
- [ ] 是否给出 before/after 验证方式

## 4. 结论建议

### Blocking

- 明显把重活放到关键路径
- 引入高频分配或高频锁争用
- 改动可能显著恶化 P99/P999 且无验证

### Non-blocking

- 有优化空间但不影响正确性
- 建议增加 benchmark 或 profiler 证据
