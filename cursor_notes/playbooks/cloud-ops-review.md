# Cursor Playbook: 云原生 / 监控 / 运维审查

## 适用场景

- 部署与配置变更
- 指标、日志、trace 调整
- 告警、容量、健康检查改动
- 故障演练和恢复设计

## 推荐提示词

```text
+先不要修改代码。
+请从可部署性、可观测性和运维角度审查当前实现：
+1. 启动 / 配置 / 依赖要求
+2. 健康检查与故障恢复
+3. 指标、日志、trace 覆盖
+4. 告警与 SLO 影响
+5. 容量与资源风险
+6. 需要补充的 runbook / 文档
+```
+
## 必查项
+
+- 默认配置是否可安全启动
+- 失败是否可诊断
+- 指标是否能解释错误率、延迟、饱和度
+- 是否有健康检查、超时、熔断、重试边界
+- 是否有回滚和演练说明

## 实战示例

### 场景：交易系统灰度发布审查

```text
先不要修改代码。
请从运维角度审查交易系统的灰度发布方案：
1. 新旧版本的配置差异是否明确
2. 灰度流量切换机制（按用户/品种/比例）
3. 关键监控指标：成交率、拒单率、延迟变化
4. 回滚触发条件和预计回滚时间
5. 数据一致性验证方案
6. 需要补的 runbook 和告警规则
```
