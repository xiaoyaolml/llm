# Cursor Playbook: 风险 / 数据 / 协议 / API 审查

## 适用场景

- 风险计算逻辑
- 市场数据接入与清洗
- 协议编解码
- API 结构调整
- 回测 / 模拟 / 校验逻辑

## 推荐提示词

```text
+先不要修改代码。
+请从风险、数据、协议和 API 角度审查这部分实现：
+1. 数值精度与边界条件
+2. 数据完整性、一致性、时效性
+3. 协议兼容性与错误语义
+4. API 变更影响面
+5. 回归测试缺口
+6. 最小风险修正方案
+```
+
## 必查项
+
+- rounding / overflow / underflow
+- 时间戳、价格、数量单位是否明确
+- 半包、坏包、乱序、重复消息
+- 版本升级兼容性
+- API 错误码和返回语义是否稳定
+- 数据延迟和脏数据处理

## 实战示例

### 场景：VaR 计算精度验证

```text
先不要修改代码。
请审查 var_calculator.cpp 的风险计算逻辑：
1. 蒙特卡洛模拟的随机数生成器质量
2. 收敛性：当前迭代次数是否足够
3. 浮点精度：中间结果累加是否有精度损失
4. 边界条件：空头寸组合、极端市场情景
5. 与基准值的比对方式
6. 回归测试覆盖
```

### 场景：FIX 协议解析升级

```text
先不要修改代码。
目标：审查 FIX 4.4 解析器的改动。
请检查：
1. 新增消息类型（35=X）的字段解析是否完整
2. Tag 缺失、重复、乱序的容错处理
3. CheckSum 校验是否覆盖所有路径
4. 半包/粘包边界是否正确
5. 与 FIX 4.2 的向后兼容性
6. 性能影响：新增拷贝或分配
```
