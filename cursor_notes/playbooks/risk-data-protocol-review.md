# Cursor Playbook: 风险 / 数据 / 协议 / API 审查

## 适用场景

- 风险计算逻辑
- 市场数据接入与清洗
- 协议编解码
- API 结构调整
- 回测 / 模拟 / 校验逻辑

## 推荐提示词

```text
+先不要修改代码。
+请从风险、数据、协议和 API 角度审查这部分实现：
+1. 数值精度与边界条件
+2. 数据完整性、一致性、时效性
+3. 协议兼容性与错误语义
+4. API 变更影响面
+5. 回归测试缺口
+6. 最小风险修正方案
+```
+
## 必查项
+
+- rounding / overflow / underflow
+- 时间戳、价格、数量单位是否明确
+- 半包、坏包、乱序、重复消息
+- 版本升级兼容性
+- API 错误码和返回语义是否稳定
+- 数据延迟和脏数据处理
