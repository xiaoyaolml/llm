# Cursor Playbook: 代码审查

## 适用场景

- 审查 PR / 分支差异
- 审查单个模块改动
- 审查回归修复

## 推荐用法

先让 Cursor 不改代码，只做审查：

```text
+先不要修改代码。
+请审查当前改动，并按以下结构输出：
+1. 正确性问题
+2. 安全问题
+3. 并发与生命周期问题
+4. 性能问题
+5. 可维护性问题
+6. 测试缺口
+7. 严重级别排序
+```
+
## 审查顺序
+
+1. 正确性
+2. 崩溃 / 未定义行为 / 安全问题
+3. 并发与资源生命周期
+4. 协议 / 接口兼容性
+5. 性能与可观测性
+6. 风格与可维护性
+
## 必查项
+
+- 空输入 / 边界值 / 极端值
+- 错误路径与资源释放
+- 日志是否足够定位问题
+- public API 是否破坏兼容性
+- 测试是否覆盖新增行为
+
## 输出建议
+
+要求 Cursor 输出：
+
+- 问题位置
+- 风险原因
+- 建议修复方式
+- 是否建议阻塞合并
