# Cursor Playbook: 性能 / 延迟 / 并发审查

## 适用场景

- 热路径优化
- 尾延迟升高
- 并发瓶颈排查
- 网络 / 数据库 / 内存热点分析

## 推荐提示词

```text
+先不要修改代码。
+请分析这个模块的性能与延迟风险，并按以下结构输出：
+1. 热路径
+2. CPU / 内存 / 锁 / I/O / 网络瓶颈
+3. 额外分配与拷贝点
+4. 队列积压与背压风险
+5. tail latency 风险
+6. 最小侵入优化方案
+7. benchmark / profiling 验证方案
+```
+
## 审查原则
+
+- 不凭感觉认定瓶颈，优先要求证据
+- 区分平均延迟与 P99/P999
+- 区分业务链路与基础设施链路
+- 区分一次性优化与长期可维护优化
+
## 必查项
+
+- 锁持有时间
+- 批处理策略
+- 缓存局部性
+- false sharing
+- 临时对象与格式化开销
+- 序列化 / 反序列化成本
+- 网络往返与重试放大
+- 数据库慢查询与连接池饱和
